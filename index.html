Project Summary

It has been widely reported that violent crime has been on the rise nationwide; a 5% increase between 2019 and 2020, according to FBI data. At a county level, violent crime rates are significantly impacted by the violence taking place in the country’s most dangerous cities. The intent of this project is to explore crime trends overall at the county level throughout the state of Texas. We utilized millions of reported offenses published by the FBI’s Crime Data Explorer between 2017 and 2 020 to discover what variables may help us predict the likelihood that a particular type of crime may take place. We considered a variety of factors including arrestee demographics, temporal and seasonality trends, as well as considered locational characteristics of where an offense occurred. While there is certainly room for improving our machine learning models and adjusting the variables we initially considered, we have been able to gain new insights regarding overall crime trends in Texas.

Data Processing

Several data processing steps were required to prepare the raw FBI data files for machine learning and visualizations after conducting the initial logical data model. Raw csv files were read into Jupyter Notebook and multiple years’ worth of data were concatenated together. Columns in the new data frames that would not be used in our analysis were removed as well as any record that was populated with “unknown” or NaN values. We merged data frames containing information about the arrestee, the incident, the offense, suspect using (under the influence), location of incident, and arresting agency to generate a consolidated data frame with all of the variables we wanted to consider for our analysis. Lastly, we converted all categorical fields to numeric values to enable us to run various machine learning models on our data.

Data Exploration and Initial Analysis

Spatial Trends

Larger population sizes do not always yield more reported arrests


The proportion of arrests by crime type vary slightly based on the location; however, crime against society represents the largest proportion of arrests made in our sample data.


The NIBRS database where we obtained our data does not contain arrest related information for all of the counties in Texas; the lack of comprehensive coverage is unknown at this time.

Seasonal and Temporal Trends

All three crime type categories follow a similar seasonal trend with fairly little variability from month to month.


An interesting pattern does appear to be emerging in April (4), when the crime trends appear to be on a steady incline but take an unexpected dip prior to shooting back up in May (5).


Temporally, the Crime Against categories of person and scoiety follow similar trends, peaking in the late night/early morning hours. The property category varies in that it peaks in the late afternoon, evening hours.

Probable Multicollinearity Between Some Independent Variables


From the coefficient heatmap we can see that there is likely a high correlation between the suburban area, population, and total police department employees’ independent variables. This makes sense given that more people live in urban areas and larger populations require larger police departments. We can remove two of these three variables for the purpose of our analyses.

Measuring Additional Correlation Between Variables

Offense Type has a strong, negative correlation with Crime Against Type; however, this variable must be removed because it has a hierarchal relationship with our dependent variable.

What is Machine Learning?

Machine Learning is the field of study that gives computers the capability to learn without being explicitly programmed - Source: Geeks for Geeks


Machine learning can be classified into one of four buckets: supervised, semi-supervised, unsupervised and reinforcement. In this analysis, we chose to use supervised machine learning models. Supervised machine learning models work well with the data selected because we have a large amount of data to train and test. Testing and Training the data is a big part of supervised machine learning because the computer uses its learning to generate a general rule that maps the inputs and outputs.The data is broken partially into training to train the model and partially into testing to test the accuracy of the model. Supervised machine learning can be broken down further into regression and classification models. In this analysis we focused on multi-class classification models, which are great at training the model to predict which classification the dependent variable is.



Our Model



Our Machine learning models were trained and tested to predict the type of crime against (independent variable) going to be committed in Texas from the years 2017-2020 based on time of day, month of the year, city type, and attributes of the arrestee(dependent variables).

Crime Against is broken into 3 categories: person, property and society, which corresponds with the offense type field. Having three categories in the independent variable, we start with the standard of trying to beat 33%. Most of the time, the more categories in the independent variable, the harder it is to have a high accurracy score. When looking at if a machine learning model is performing well, it is important to look at the accuracy score, precision, recall and f1-score. Below we will dig into different 4 different models and there plus and minuses for this analysis.

Results


Machine Learning Results

Random Forest Model

The Random Forest Model builds multiple decision tree algorithms and has each decision tree input its vote. The majority answer becomes the predicted outcome. This is a great model for combining weak learners and turning it into a strong model.

The Random Forest Model had the best scores of any of the models.

  • Accuracy Score = 73%
  • Precision = 73%
  • Recall = 73%
  • F1-score = 73%

Category 3 had the highest precision score (77%) and highest recall score (82%). This makes logical sense as there this category had the most crimes in total.

With this model one thing to keep in mind is that it is built for numerical data. We have converted all the categorical data into numeric data but there is still the potential risk of weighting variables to0 highly because the data is categorical utilizing the random forest model.

Multinomial Model

The Multinomial Model is logistic regression model used for multiclassification kind of problems. It is utilized to predict when the target class is more than two.--The Multinomial logistic regression workflow/stages - Source: dataaspirant.com

  • Inputs
  • Linear model
  • Logits
  • Softmax Function
  • Cross Entropy
  • One-Hot-Encoding

One-Hot Encoding like get_dummies is an important part of supervised machine learning. The function breaks out categorical data into columns to help with even weights of the data run through the model.

Multinomial data came up with an accuracy score of 57%. A respectable score when working with multi-class classification models.

Random Over Sampler

The random over sampler model helps with class imbalance. It does this by randomly resampling the training data through duplicating samples from the minority class. In the FBI crime data, there is an imbalance in the crimes against classes. When running this model, it was able to beat probability, but the accuracy score lagged behind at 41% compared to the other models.

SMOTE Oversampling

Like the random over sampler model, the smote oversampling model duplicates values from the minority class to balance the distribution. SMOTE is short for synthetic minority oversampling technique. Also like random over sampling, the SMOTE over sampling model only achieved a 41% accuracy score. In order for this model to perform better, it is best to have a larger disparity in the data distribution.
