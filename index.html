Research Question

Can we predict the type of of crime that will take place based on a variety of spatial, temporal and categorical factors?

Dependent Variable: Crimes Against Type (property, society, person)

Inependent Variables:

Incident Month

Incident Hour

Suburban Area Flag

Location Type of Crime

Arrestee Age

Arrestee Gender

Suspect Using

Data Exploration and Initial Analysis

Visualizing Spatial Trends

Larger population sizes do not always yield more reported arrests


The proportion of arrests by crime type, vary slightly based on location; however, Crime Against Category 3 appears to represent the largest proportion for most counties.


The NIBRS database does not contain data for several Texas counties; the lack of coverage is currently unknown.

Visualizing Seasonal and Temporal Trends

There appears to be little variability in seasonal trends, likely due to mild seasonal changes in the weather throughout most of Texas


Temporally, the Crime Against categories of person and scoiety follow similar trends, peaking in the late night/early morning hours. The property category varies in that it peaks in the late afternoon, evening hours.

Probable Multicollinearity Between Some Independent Variables

The p value for the suburban area flag 1 and 2 is significantly low (<0.0001)


The r squared value for the suburban area flag 1 and 2 is within .02 of each other; .99 and .97 respectively


Given the statistically significant correlation, we can remove two of the 3 independent variables for the purposes of our multinomial logistic regressionâ€”using the binary variable for the suburban area flag will make our machine learning more efficient because it will not need to evaluate many discrete, integer values (population and total employees)

Measuring Correlation Between Variables

Offense Type has a strong, negative correlation with Crime Against Type; however, this variable must be removed because it has a hierarchal relationship with our dependent variable.


The high correlation between Population, Total Employees (police department) and Suburban Area Flag is visible in our heat map.

Database Design and Storage

Our team utilized Amazon Web Services (AWS) to host our raw and processed data. We created a S3 Bucket to store our unstructured and reference content. The data required for initial data analysis and machine learning modeling was published to a postgres database and queried using SQLAlchemy to run our models. Additionally, SQL queries and joins were conducted to building relationships within the database between multiple tables.
